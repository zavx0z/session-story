# OCR Findings

OCR рассматривался как первый способ получить таймлайн из видео.

Идея была понятной:

```text
video -> frames -> OCR -> text timeline -> voiceover script
```

Но для записи AI-сессии этот путь оказался плохим как основной.

## Что пробовали

Нарезка кадров:

```bash
ffmpeg -i interpret_4min_final_2.mp4 \
-vf "fps=2,scale=1920:-1,format=gray,eq=contrast=1.45:brightness=0.03,unsharp=5:5:1.0" \
ocr_frames/frame_%06d.png
```

OCR через Tesseract:

```bash
tesseract frame.png stdout -l rus+eng --psm 6
```

Также пробовался режим только `rus`, потому что сначала был установлен только русский языковой пакет.

## Почему результат плохой

OCR по всему экрану плохо подходит для этого материала:

- текст слишком мелкий;
- часть текста на английском;
- много кода и имён файлов;
- интерфейс состоит из нескольких областей;
- есть терминал, браузер, редактор, чат, панели;
- тёмная тема ухудшает распознавание;
- OCR не понимает действие, он только пытается читать буквы;
- 241 OCR-сегмент на короткое видео даёт слишком много шума.

## Когда OCR может быть полезен

OCR можно оставить как fallback:

- для одной заранее обрезанной области экрана;
- для крупного текста;
- для терминального вывода;
- для проверки конкретного таймкода;
- для поиска видимой ошибки на экране.

Но не как основной способ построения сюжета.

## Что лучше

Для основного анализа лучше использовать:

```text
video -> frames/contact sheets -> Qwen Vision -> factual timeline -> GPT story
```

Qwen Vision может увидеть:

- кто действует;
- где Codex пишет код;
- где идёт запуск команд;
- где меняется интерфейс;
- где просто ожидание;
- какие кадры важны для зрителя.

## Вывод

OCR не удаляется из картины полностью, но переводится в статус вспомогательного инструмента.

Основной путь проекта — визуальный анализ через Qwen.
